client = anthropic.Anthropic(
    base_url="http://localhost:8080",
    api_key="placeholder",
    default_headers={"X-Tenant-ID": "tenant_ecommerce"},

    model="claude-opus-4-5",
    max_tokens=1024,

LLM

Requires X-Tenant-ID header.

POST /v1/chat/completions OpenAI-compatible chat. Pass stream: true to receive token-by-token SSE, or stream: false for a single JSON response.
POST /v1/messages Anthropic Messages API. Accepts model, messages, max_tokens. Pass stream: true for SSE with content_block_delta events, or stream: false for a single JSON response.

MCP

Requires X-Tenant-ID header. Implements JSON-RPC 2.0 over HTTP.

POST /mcp/message MCP JSON-RPC endpoint. Pass Accept: text/event-stream to wrap the response as an SSE event (spec 2025-03-26).
GET /mcp/sse Legacy HTTP+SSE transport (spec 2024-11-05). Emits an endpoint event with the POST URL, then pings every 15 s.

Supported methods

tools/list Returns all tools for the tenant. Tenant tools first, then global tools (unless include_global_tools is false in tenant settings).
tools/call Calls a named tool. Returns mock JSON from the matching tenant scenario, falling back to the global registry.
resources/list Returns an empty resource list (stub).
prompts/list Returns an empty prompt list (stub).
initialize Returns server info and capability advertisement for the 2025-03-26 protocol version.

State override

Each tenant has a JSON state store. When tools/call matches a built-in scenario, it checks whether state[scenario.state_key] exists — if so, that value replaces the default mock payload. Lets you inject custom data per-tenant without writing new scenarios. 

curl -s -X POST http://localhost:8080/mcp/message \
  -H "X-Tenant-ID: saas-demo" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'

# Returns all tools available to this tenant.
# Built-in: get_growth_metrics, get_churn_report, … (saas domain)
# Plus any tools you added in the Playground → Tenants → MCP Tools tab.
curl -s -X POST http://localhost:8080/mcp/message \
  -H "X-Tenant-ID: saas-demo" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":2,"method":"tools/call",
       "params":{"name":"get_growth_metrics"}}'

# Returns the built-in mock payload:
# { "mrr_usd": 128400, "churn_rate_pct": 2.1, "nrr_pct": 118 }
# Write custom data under the scenario's state_key ("growth_metrics")
curl -s -X PUT http://localhost:8080/admin/tenants/saas-demo/state \
  -H "Content-Type: application/json" \
  -d '{
    "state": {
      "growth_metrics": {
        "mrr_usd": 999,
        "churn_rate_pct": 0.1,
        "nrr_pct": 150
      }
    }
  }'

# Or set it visually: Playground → Tenants → State tab → Save state
# Same call as step 2 — now returns YOUR data
curl -s -X POST http://localhost:8080/mcp/message \
  -H "X-Tenant-ID: saas-demo" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","id":3,"method":"tools/call",
       "params":{"name":"get_growth_metrics"}}'

# Returns your overridden payload:
# { "mrr_usd": 999, "churn_rate_pct": 0.1, "nrr_pct": 150 }

# To reset: PUT state back to {} or remove the key.

State keys per built-in scenario: growth_metrics top_churned_accounts incidents deployments inventory security_events matters and more — visible in the Scenarios tab under each built-in entry.

Admin

Unauthenticated — local dev only. No X-Tenant-ID required.

Tenants

GET /admin/tenants List all tenant IDs.
POST /admin/tenants Create a tenant. Body: {"tenant_id":"…","state":{}}
GET /admin/tenants/{id} Get tenant state JSON.
PUT /admin/tenants/{id}/state Replace tenant state. Body: {"state":{}}
GET /admin/tenants/{id}/settings Get tenant settings (e.g. include_global_tools).
PATCH /admin/tenants/{id}/settings Merge-patch tenant settings. Body: {"include_global_tools":false}
DELETE /admin/tenants/{id} Delete tenant and all its scenarios.

Scenarios

GET /admin/tenants/{id}/scenarios List all scenarios for a tenant.
POST /admin/tenants/{id}/scenarios Add a scenario. Body: {"keywords":["…"],"response":"…","tool_name":"…","tool_data":{}}
DELETE /admin/tenants/{id}/scenarios/{sid} Delete a scenario by ID.

Chaos

GET /admin/tenants/{id}/chaos Get current chaos profile.
POST /admin/tenants/{id}/chaos Set profile: none rate_limit server_error latency
