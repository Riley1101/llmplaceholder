How it works

01

Pick a tenant

Each tenant has its own mock state and scenario library. Pass X-Tenant-ID in every request header.

02

Send a message

Use the OpenAI-compatible /v1/chat/completions endpoint or send MCP JSON-RPC to /mcp/message.

03

Get mock data

Responses stream token-by-token from a scenario matched by keyword. Inject chaos to simulate 429s, 500s, or latency.

Routes

LLM endpoints require X-Tenant-ID header. Admin endpoints are unauthenticated — local dev only.

LLM

POST /v1/chat/completions OpenAI-compatible chat — streams SSE. Pass stream: true for token-by-token output.
POST /mcp/message MCP JSON-RPC 2.0 — supports tools/list, tools/call, resources/list.

Tenants

GET /admin/tenants List all tenant IDs.
POST /admin/tenants Create a tenant. Body: {"tenant_id":"…", "state":{}}
GET /admin/tenants/{id} Get tenant state JSON.
PUT /admin/tenants/{id}/state Replace tenant state. Body: {"state":{}}
DELETE /admin/tenants/{id} Delete tenant and all its scenarios.

Scenarios

GET /admin/tenants/{id}/scenarios List all scenarios for a tenant.
POST /admin/tenants/{id}/scenarios Add a scenario. Body: {"keywords":["…"],"response":"…"}
DELETE /admin/tenants/{id}/scenarios/{sid} Delete a scenario by ID.

Chaos

GET /admin/tenants/{id}/chaos Get current chaos profile for a tenant.
POST /admin/tenants/{id}/chaos Set profile: none rate_limit server_error latency
